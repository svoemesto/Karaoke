## TL;DR

Модель монетизации (75% free / 25% premium) — **вторичный** приоритет, решаем позже
после роста базы. Trial не делаем в первом раунде (anti-fraud дороже выгоды).

## 3. Принципы

1. **Visitor→registration — главный фокус** в первом раунде (потенциал ×5).
2. **Модель монетизации — вторичный приоритет** (сначала вырастить базу).
3. **Trial не делаем** — anti-fraud дороже выгоды при текущей базе.
4. **Сначала QW (быстрые победы), потом M (среднесрочные)** — измерить эффект на каждом этапе.
5. **Каждое изменение — с метриками** (M-1, M-2: воронка в `tbl_events`).
6. **Не ломаем обратной совместимости** — все изменения аддитивные, opt-in.
7. **Только онлайн** (см. оферту) — НЕ упоминать MP4/скачивание в рекламных материалах.
   Доступ — для личного некоммерческого использования в браузере.

## 4. Принятые решения

### 4.1. Модель монетизации

**Решение**: гибридная (модель D из аудита):
- **Эфир-N-дней** (например, 14 дней) — для новых песен
- **«1 трек с каждого исполнителя» free навсегда** — «визитная карточка» (для анонимов)
- **«Первый альбом у каждого исполнителя» free навсегда** — для зарегистрированных (бонус)
- **Остальное — только премиум**

**Но!** В первом раунде **не реализуем** — сначала M-15 (анализ `tbl_settings.album`), потом A/B-тест.

### 4.1.1. Стратегия площадок (акцент только на сайте) — НОВОЕ

**Решение**: переходим на **сайт-центричную модель**.
- **На сайте** — вся коллекция (по подписке). Акцент в рекламных материалах — **только на сайт**.
- **Сторонние площадки** (Sponsr, Dzen, VK, Max, Telegram) — **технический канал** для
  DEMO-публикаций. **В рекламных материалах не упоминаются** (акцент только на сайте).
- **Уже опубликованные на площадках полные версии** — пока не трогаем, возможно удалим позже.

**Импликации**:
- `telegram-auto-publish.md` — нужна перенастройка: публиковать DEMO вместо full-версии
- Расписание публикаций (5 площадок) — пересмотреть
- Sponsr — **особый случай**: там оформляется подписка, можно оставить более полный контент
- Трафик с площадок: ожидаем **просадку** (раньше приходили за бесплатным, теперь — за демо
  и переходом на сайт)
- Главный канал привлечения теперь — **органика + прямой трафик + brand-awareness площадок**,
  а не «бесплатный контент ради трафика»
- **Никаких ссылок на соцсети в рекламных материалах** (страница «О проекте», главная, /premium)

**Обратная сторона**: меньше вирального роста через соцсети, но больше — за счёт полной
коллекции на сайте (а не огрызков в каждой соцсети).

### 4.2. Судьба 11 097 уже эфирных

**Решение**: «решим позже». В первом раунде **grandfather** (не трогаем), чтобы не потерять трафик.
Стратегия их не трогает, они остаются free.

### 4.3. Trial

**Решение**: **пока без trial**. QW-3, M-11, T1.x-T3.x — в backlog.
Anti-fraud нужен только если/когда добавим trial.

### 4.4. Anti-fraud (от sharing)

**Решение**: можно делать **параллельно** (не зависит от trial).
- Минимум: S1.1 (max 2 сессии) + S1.3 (список в кабинете) — 3-4 недели.
- B1 «Семейный» тариф — как **легальная альтернатива** sharing.

### 4.5. Тарифы

**Решение**: текущие тарифы остаются (449 ₽/мес, 49 ₽/песня бессрочно).
**Новые тарифы**:
- **M-4** — годовой с -30% (449×12×0.7 = 3 770 ₽/год)
- **M-21** — «семейный» (3-5 устройств за 2x цену) — если B1 делаем

### 4.6. Ценообразование (что обсудить позже)

- Бесплатный trial (отложен)
- Подарочный тариф (отложен)
- Корпоративный (отложен)

### 4.7. «Мега-премиум» — отложенная концепция

**Решение**: аккорды и табы в онлайн-плеере, расширенная навигация по структуре песни,
прочие «продвинутые» фичи — **отложены** и будут позиционироваться как отдельный
уровень подписки («мега-премиум»), а не как стандартная часть премиума.

**Почему**:
- Аккорды и табы делаются **далеко не для всех песен** (трудоёмко, требует
  ручной/полуавтоматической работы) — если включить в обычный премиум, пользователь
  будет ожидать их для **каждой** песни, а получит для 5-10% каталога
- Переключение тональности в онлайн-плеере — ещё не реализовано
- Расширенная навигация по структуре — вторичная фича

**Когда делать**: во втором раунде, после стабилизации visitor→registration и
базового премиума. Возможно, как отдельный тариф (выше обычного премиума по цене)
или как опция для отдельных песен.

**Не рекламируем** эти фичи в QW-9 (страница «О проекте») и других рекламных
материалах — чтобы не создавать завышенных ожиданий.

## 7. Метрики и контроль

**Трекинг** (уже есть в `tbl_events`):
- `engagement` (время на странице) — есть
- `player` (действия в плеере) — есть
- `ui` (навигация, тема, скролл) — есть
- `callRest` (API-вызовы) — есть

**Чего не хватает** (добавим в M-2):
- `event_type='register'`
- `event_type='login'`
- `event_type='subscription_paid'`
- `event_type='subscription_canceled'`

**Воронка** (добавим в M-1, дашборд в webvue3):
- визит → регистрация → премиум
- по источнику трафика
- по исполнителям/категориям

**Retention** (добавим в M-1):
- D1/D7/D30 для премиум-подписчиков
- churn rate по месяцам
